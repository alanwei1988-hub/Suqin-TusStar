# 另一台电脑部署指南

这份文档用于说明：把仓库拉到另一台电脑后，怎样完成最小配置并正常启动。

## 1. 前置要求

- Node.js 18+
- Python 3
- Git
- 可访问模型服务
- 企业微信智能机器人已拿到 `BOT_ID` 和 `SECRET`

说明：

- 项目包含 Node.js 依赖，也包含 `markitdown` 的 Python 运行时。
- 执行 `npm install` 时，会自动创建项目内的 Python 虚拟环境并安装附件解析依赖。

## 2. 拉取仓库

```powershell
git clone <你的仓库地址>
cd wxwork-bot
```

## 3. 创建 `.env`

根目录新建 `.env` 文件，内容如下：

```env
BOT_ID=你的企业微信机器人ID
SECRET=你的企业微信长连接密钥
OPENAI_API_KEY=你的主模型密钥
MARKITDOWN_OCR_OPENAI_API_KEY=你的OCR回退模型密钥
DASHSCOPE_API_KEY=你的阿里百炼OCR密钥
```

当前项目按这 5 个环境变量即可启动和运行。

## 4. 检查 `config.json`

第一次在新电脑部署时，建议先确认这几个配置：

### 4.1 主模型配置

当前仓库默认配置：

- `agent.model`: `gpt-5.4`
- `agent.openai.baseURL`: `https://api.zetatechs.com/v1`

所以 `.env` 里不需要再重复写 `MODEL_NAME` 和 `OPENAI_BASE_URL`。

### 4.2 OCR 配置

当前仓库默认配置：

- `activeLlmProfile`: `qwen-vl`
- `fallbackLlmProfile`: `legacy-openai-compatible`

因此建议同时保留两套 OCR 密钥：

- `DASHSCOPE_API_KEY`
- `MARKITDOWN_OCR_OPENAI_API_KEY`

如果你以后明确不需要回退链路，也可以只保留当前实际启用链路对应的密钥。

### 4.3 合同归档目录

当前仓库里的 `contractMcp.libraryRoot` 还是本地测试目录：

```json
"libraryRoot": "storage/已签署协议电子档"
```

如果新电脑要接真实 NAS 或共享盘，启动前把它改成目标机器可访问的实际路径。

如果只是本地调试，可以保持不变。

## 5. 安装依赖

在项目根目录执行：

```powershell
npm install
```

安装过程中会自动执行：

```powershell
npm run markitdown:install
```

它会完成这些事：

- 创建 `.tools/markitdown` Python 虚拟环境
- 安装 `markitdown`
- 安装 `markitdown-ocr`
- 安装 `openai`

如果自动安装失败，先检查：

- `python` 或 `py -3` 是否可用
- `pip` 是否可正常联网

然后手动重试：

```powershell
npm run markitdown:install
```

## 6. 启动项目

```powershell
npm start
```

启动成功后，程序会建立企业微信长连接并开始收消息。

## 7. 验证是否正常

建议按下面顺序验证：

1. 先看控制台是否正常启动，没有报环境变量缺失或 Python 安装失败。
2. 在企业微信里给机器人发一条普通文本，确认能收到回复。
3. 发一个 PDF 或 Office 附件，确认附件解析链路可用。

如需跑测试：

```powershell
npm test
```

如需单独验证 OCR：

```powershell
npm run test:markitdown-ocr
```

## 8. 常见问题

### 8.1 两台电脑能不能同时跑同一个机器人？

不能。

同一个企业微信机器人同一时间只能保留 1 个有效长连接。新电脑连上后，旧电脑上的连接会被踢掉。

### 8.2 为什么仓库拉下来后没有 `.env`、`storage`、`data`、`.tools`？

因为这些都属于本地运行时文件，已经被 `.gitignore` 忽略，不会提交到远程仓库。

### 8.3 如果只想本地调试，不接真实 NAS，可以吗？

可以。

保持 `contractMcp.libraryRoot` 为：

```json
"libraryRoot": "storage/已签署协议电子档"
```

项目会在本地目录下创建归档目录和数据库。

## 9. 最小部署清单

新电脑上至少完成以下事项：

1. 安装 Node.js 18+
2. 安装 Python 3
3. `git clone` 仓库
4. 在根目录创建 `.env`
5. 确认 `config.json` 中的 `libraryRoot` 是否要切到真实 NAS
6. 执行 `npm install`
7. 执行 `npm start`
